        HTMLFormElement - Tag 'form'

        Absenden von Daten wird durch Event 'submit' getriggert
            vorm Absenden werden alle Inputs geprüft

        Beachte: Buttons sind per default vom Typ 'submit'.
        Beachte: zwei Tags für Buttons - button und input. 

        Action beim 'form' - beinhaltet url zu der Datei, die die Daten vom Formular bearbeiten muss.

        Inputs haben Eigenschaft 'validity' oder 'valid'.
            ValidityState {valueMissing: false, typeMismatch: false, patternMismatch: false, tooLong: false, tooShort: false, …}
                badInput: false
                customError: false
                patternMismatch: false
                rangeOverflow: false
                rangeUnderflow: false
                stepMismatch: false
                tooLong: false
                tooShort: false
                typeMismatch: false
                valid: true
                valueMissing: false

        Hidden Inputs werden genutzt, um durch die Daten zum Server zu schicken, die der Benutzer nicht unbedingt sehen muss.
        Diese Daten werden dynamisch in die Hidden-Inputs reingeschrieben.
        Warum schickt man die Daten zum Server über Hidden Inputs und nicht durch JSON, XML etc?
        Weil Json zu bauen ist ein extra Schritt. Und alle Einträge aus Inputs (auch von hidden) werden durch die URL an den Server geschickt.
        URL Beispiel: _urlVomForm_/?_key_=_value&_key_=_value&.....
